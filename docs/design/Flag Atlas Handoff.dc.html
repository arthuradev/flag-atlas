Direção Terrain — implementação React + Tailwind

Guia de implementação do design system “Modern cartographic learning game”. Base Terrain (sálvia + verde-terreno) com detalhes de atlas de Meridian (textura de curvas de nível e ocre/dourado). Tudo offline: ícones Lucide (via lucide-react, tree-shaken), bandeiras vetoriais e Nunito auto-hospedada — sem CDN obrigatório.

1 · Fundamentos da direção

Superfícies planas e limpas (Terrain) com uma camada cartográfica sutil: fundos de curva de nível em radial-gradient repetido, ocre reservado para XP / ofensiva / recompensas, e a bandeira sempre como protagonista (moldura branca + sombra macia no mobile, full-bleed no desktop). Evite: gradiente roxo/azul, glassmorphism, emojis como ícones de UI, e o layout mobile centralizado no desktop — o desktop usa sidebar fixa + grid de dashboard.

4 · Tipografia, espaçamento, raio & sombra

Fonte única: Nunito, auto-hospedada (@fontsource/nunito, pesos 600/700/800/900). Escala e pesos:

Display30–44px · 900 · tracking-tightheaders de tela, resultado
H222–28px · 800/900pergunta, títulos de card
Body15–16px · 600/700texto, opções
Caption11–13px · 700/800 · uppercase trackingrótulos, kickers

Espaçamento — escala 4px (Tailwind padrão): gap de UI gap-2/3, padding de card p-5/6, seções gap-5. Raio — chip 10 · btn 14 · card 20 · xl2 28 · full (pills/avatares). Sombra — shadow-card em superfícies, shadow-btn (offset sólido) nos botões primários, shadow-flag na bandeira protagonista.

DomainBadge — bronze / prata / ouro / platina

Insígnia = escudo hexagonal (clip-path) com gradiente metálico + ícone Lucide (Shield / ShieldCheck / Gem na platina). O tier vem dos limiares de mastery do repo (ver §7).

const TIER = {
  bronze:   {grad:'from-[#C88A52] to-[#96602F]', ink:'#F3E6D4', Icon:Shield},
  silver:   {grad:'from-[#CED6DB] to-[#98A3AA]', ink:'#3F4A50', Icon:Shield},
  gold:     {grad:'from-[#ECC158] to-[#C1901F]', ink:'#5A4413', Icon:ShieldCheck},
  platinum: {grad:'from-[#7FD7FF] to-[#34C07D]', ink:'#0F2B22', Icon:Gem},
};
const hexClip = {clipPath:'polygon(50% 0,100% 24%,100% 70%,50% 100%,0 70%,0 24%)'};

function DomainBadge({tier='gold', size=64}) {
  const t = TIER[tier];
  return (
    <span style={{...hexClip, width:size, height:size*1.16}}
      className={`bg-gradient-to-br ${t.grad} inline-flex items-center justify-center`}>
      <t.Icon strokeWidth={2} color={t.ink} width={size*0.42}/>
    </span>
  );
}

6 · Ícones & bandeiras (offline)

Ícones: lucide-react — importe só o que usar (tree-shaking). Padrão: strokeWidth 2, size 18–24, color currentColor. Mapa usado nas telas: Home, Layers (coleção), Trophy, BarChart3, ShoppingBag, Settings2, Compass (logo), Play, Flame (ofensiva), Sparkles (XP), Target (precisão), CheckCircle2, XCircle, Shield/ShieldCheck/Gem (badges), Lock, Search, Coins, Moon, Volume2, Bell. Zero emojis na UI.

Bandeiras: use SVGs vetoriais empacotados no bundle (ex. flag-icons via npm, ou os 195 SVGs no public/flags/) — nunca um CDN. Renderize com rounded-lg overflow-hidden shadow-flag. No treino (mobile) a bandeira fica emoldurada em branco; no desktop, full-bleed.

8 · Acessibilidade

  • Contraste: pares de token atendem WCAG AA (ink/surface ≈ 13:1; pine/branco ≈ 4.8:1). Nunca texto faint sobre bg em tamanhos < 14px.
  • Foco visível: aplique um utilitário global focus-visible:ring-2 focus-visible:ring-pine focus-visible:ring-offset-2 ring-offset-bg em todos os alvos interativos.
  • Reduced motion: @media (prefers-reduced-motion: reduce){ *{animation:none!important;transition:none!important} } — e o toggle de Configurações força a mesma classe.
  • Alvos de toque: mínimo 44×44px (opções, itens de nav, FAB). Estados de resposta comunicam por cor e ícone (CheckCircle2/XCircle), nunca só por cor.
  • Semântica: quiz como <fieldset>/<legend>, opções como <button role="radio">; aria-live="polite" no feedback de resultado.

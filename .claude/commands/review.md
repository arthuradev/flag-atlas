# /review

Use este comando para revisar a fase antes de avançar.

## Checklist

- A implementação respeita o MVP?
- Há lógica complexa presa em componente React?
- Os testes cobrem a lógica nova?
- O app continua responsivo?
- A UX está clara?
- Acessibilidade básica foi considerada?
- Documentação foi atualizada quando necessário?
- O commit está pequeno e claro?

## Não aceitar

- Build quebrado.
- Testes removidos para passar.
- Features fora do MVP.
- Código sem tipagem adequada.
- UI funcional mas visualmente descuidada.

# /plan

Use este comando para planejar a próxima fase.

## Procedimento

1. Leia `.specs/TASKS.md`.
2. Identifique a fase atual.
3. Liste objetivos, arquivos prováveis e riscos.
4. Confirme que a fase não adiciona features fora do MVP.
5. Defina validações a rodar antes do commit.

## Saída esperada

- Fase atual.
- Objetivo.
- Tarefas.
- Arquivos principais.
- Testes/validações.
- Commit sugerido.
